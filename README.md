Для запуска тестов зайти в контейнер fastapi-app через bash:
```bash
/test.sh
```

Для обеспечения идемпотентности задача confirm_booking в начале выполнения проверяет текущий статус брони. Если статус уже является финальным (CONFIRMED или FAILED), выполнение немедленно завершается с соответствующим результатом, минуя бизнес-логику и имитацию сбоев. Такой подход гарантирует, что повторные вызовы с тем же booking_id не изменят состояние системы и не вызовут побочных эффектов (например, повторной отправки уведомлений или изменения статуса).
